## Welcome to Exercise A

In this guided exercise you will work with the three fundamental building blocks of every Kubernetes production workload:

- A **Deployment** that manages your application replicas
- A **ReplicaSet** automatically created and managed by the Deployment
- A **Service** that provides a stable network endpoint in front of your Pods

Every step explains not just *what* to run but *why* it works that way. Read each explanation before running the command.

**Environment:** a single-node Kubernetes cluster is already running. `kubectl` is configured and ready.
