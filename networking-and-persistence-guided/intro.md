# Networking and persistence — Guided track

**Cloud Computing and Distributed Systems**
Università degli Studi di Verona — A.A. 2025/2026

---

## What you will do in this exercise

You will work with a small Python web application that counts visits and saves the count to a file. Through a series of guided steps, you will discover why container data disappears by default — and how to fix it using Docker volumes. You will then connect two containers to each other over a Docker network so they can communicate by name, and you will learn the difference between declaring a port and publishing it to the host.

You do not need to understand the Python code. Treat the application as a black box — your focus is entirely on Docker commands.

---

## How to follow this exercise

Each step contains:
- a brief explanation of the concept
- the exact commands to run — copy and paste them into the terminal on the right
- a question to answer before moving to the next step

Read the output of each command carefully before answering.
